Usage playbook

  • Copy the exact address string from BrokerBay search so the assistant selects the right listing row.
  • Optionally set a preferred time like 10:00 AM; the runner behaves exactly like node auto-book-enhanced.js "Address" "10:00 AM" and will walk forward to the next available slot if that exact time is taken.
  • Keep Chrome signed in to BrokerBay before launching headless runs. Use the visible browser option if the login session expired.
  • The run writes every successful booking into src/data/data.db, so the dashboard picks it up instantly.
  • Need the script to wait for auto-confirm slots only? Toggle the AUTO-CONFIRM switch before launching.
  • Do not close the server window while a run is active‚Äîthe process log here mirrors the actual CLI output.

What happens behind the scenes

  1. The server spawns node auto-book-enhanced.js <property> [<preferred time>] [<preferred date>] [<preferred duration>] with the options you toggled here.
  2. The Puppeteer session logs in with your saved profile, searches the listing, books a showing, and hits AUTO-CONFIRM whenever possible.
  3. Every console line is streamed back into this page so you can watch progress without touching the CLI.
  4. Confirmed bookings flow straight into the shared SQLite database, which powers the dashboard next door.
